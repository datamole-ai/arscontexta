# Methodology Tradition Background

Background examples for vocabulary, processing language, and failure-risk intuition. Traditions are not selectable architectures.

Every generated vault keeps the same kernel: flat note collection, wiki links plus semantic search, topic-map navigation, templates, hooks, health checks, full pipeline, and self/ continuity.

---

## Zettelkasten

**Useful for:** Research, academic work, long-term intellectual projects, concept-heavy domains.

**Borrow:**
- Prose titles that make claims.
- Dense explicit links with context.
- Notes written in the user's own words.
- Topic maps as emergent navigation.

**Watch for:**
- Notes created but never connected.
- Over-investment in system design instead of knowledge work.

---

## PARA

**Useful for:** Project execution, task-heavy workflows, reference storage.

**Borrow:**
- Actionability vocabulary: project, area, resource, archive.
- Review language around active vs inactive work.
- Low-friction capture and routing.

**Watch for:**
- Folder groupings proposed only because they look tidy.
- Project silos that hide cross-domain connections.

---

## Evergreen Notes

**Useful for:** Writing projects, conceptual exploration, evolving understanding.

**Borrow:**
- Living notes that improve through revisiting.
- Contextual links that explain why a connection matters.
- Strong title quality standards.

**Watch for:**
- Aspirational notes that are never revised.
- Overly polished notes before enough material exists.

---

## Cornell Note-Taking

**Useful for:** Courses, learning, structured review, retention.

**Borrow:**
- Record, reduce, recite, reflect, review as processing language.
- Cue/question framing for recall.
- Summary sections that force synthesis.

**Watch for:**
- Recording without later review.
- Summaries that restate rather than test understanding.

---

## GTD

**Useful for:** Execution-heavy domains, operations, commitments, follow-up tracking.

**Borrow:**
- Capture everything, then clarify.
- Next-action language.
- Waiting-for and follow-up vocabulary.
- Condition-based review.

**Watch for:**
- Dense action metadata without a reader.
- Trust collapse when review lapses.

---

## Memory Palace

**Useful for:** Sequential recall, presentation preparation, memorization tasks.

**Borrow:**
- Vivid retrieval cues.
- Ordered walkthrough language.
- Spatial metaphors when the user's domain already uses them.

**Watch for:**
- Forcing spatial hierarchy onto non-sequential knowledge.
- Treating a mnemonic technique as a general knowledge architecture.
